# 강의록 온보딩 가이드
버전: v1_0

강의록 제작 전용 안내. 공통 온보딩(`ONBOARDING v4_0`)을 기반으로 하며, 이 문서는 강의록에만 고유한 사항을 다룬다.

---

## 버전 이력

**v1_0** (현재): `ONBOARDING v3_11`에서 강의록 전용 내용을 분리.

---

## 레포 우선 원칙

공통 온보딩의 레포 우선 원칙을 그대로 따른다. 강의록 마크다운→LaTeX 변환은 `md2tex.py`, tikz 삽입은 `insert_tikz.py`, 빌드는 `build.py`를 사용한다. 즉석 스크립트를 새로 만들지 않는다.

---

## 세팅

```bash
bash setup.sh
```

공통 세팅(폰트, luatexja, KoNLPy, master.sty, translation.sty, build.py) + 공통·강의록·번역본 규칙 문서 전체가 프로젝트 디렉터리에 배치된다.

---

## 주제 설명

주제를 줄 때는 반드시 아래 정보를 함께 제공한다.

- **어떤 텍스트**를 중심으로 다룰 것인지
- **어떤 논점**을 담고 싶은지

기본값: 맥락은 학생들을 위한 수업 자료 제작, 대상은 고전학 강의 수강생(학부 1–4학년), 논점은 개념과 텍스트 설명 중심으로 원전을 처음 접하는 독자를 전제하고 최대한 쉽고 명확하게 풀어 설명한다.

기본값에서 벗어나는 경우(심화 세미나, 특정 학년 등)에는 명시한다.

## 문서 작업 흐름

마크다운 컨펌 없이 바로 PDF 작업으로 넘어가지 않는다.

1. 마크다운 초안 작성 → 말미에 "확인이 필요한 항목" 목록 제시
2. 사용자 컨펌
3. **시각화 검토 실행** (`.tex` 작성 전 필수): `lecture_style_guide`의 "시각화 검토 규칙"에 따라 후보를 추출하고 보고 → 사용자 컨펌
4. `.tex` 파일 작성 (저술 정의, tikz 도식 이 단계에서 삽입)
5. `python3 build.py 문서이름.tex`으로 컴파일
6. 렌더링 검증 결과 확인 → 경고 있으면 처리 여부 확인
7. `output/`에 버전 번호가 붙은 PDF + TEX 저장

## tikz 도식 작업흐름

**마크다운 단계에서는 도식을 삽입하지 않는다.** tikz 코드는 `.tex` 작성 단계에서만 삽입한다.

작업 순서: 마크다운 초안 작성(마크다운 표만 사용) → 사용자 컨펌 → 시각화 검토 실행 → 사용자 컨펌 → `.tex`에 tikz 코드 삽입.

tikz 도식이 포함된 문서의 프리앰블에는 아래를 추가한다.

```latex
\usepackage{tikz}
\usetikzlibrary{calc, positioning, arrows.meta}
```

### tikz 도식 너비 계산

A4 기준 textwidth는 약 140mm. 도식 전체 너비가 140mm를 초과하면 `Overfull \hbox` 경고와 함께 우측으로 삐져나온다.

## 강의록 전용 명령어

| 명령어 | 용도 |
|--------|------|
| `\weeksection{주차}{분량}{제목}` | 주차 강의록 상단 헤딩 (14pt 네이비, 우측 정렬) |
| `\bilingualquote{원문}{번역}{출처}` | 더블 칼럼 인용 (원문 좌 55% / 번역 우 44%) |
| `\srcquote{원문}{번역}{출처}` | 단독 짧은 인용 (원문 / 구분선 / 번역 / 출처) |
| `\bqnote{내용}` | `\bilingualquote` 번역란 내 각주 |

공통 명령어(`\gr`, `\gri`, `\grb`, `\citework`, `\usework`, `\needspace` 등)는 `ONBOARDING v4_0` 참조.
